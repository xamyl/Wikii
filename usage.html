Usage

Editing

Once you have your project, start editing the index.md file in the /docs folder to add content for your wiki.

Viewing your site

Type

npm start

into your shell. This uses express, which supports live editing, so you can edit pages live!

Making extra pages

To make more pages, just make more markdown files in your /docs directory.

Building and deploying your site

First, to build your site, run

npm run build

This will convert your docs files into html files and put them in the /dist directory.

To deploy it, upload all files from the /dist directory onto the platform you want to use.

Vercel/Github Pages/Gitlab Pages/Codeberg Pages

Upload the contents of your /dist directory to your chosen git provider [Side note, for vercel, Add your git provider and repository], and follow the steps on the site to deploy it.

Netlify

Either follow the tutorial above, or drag and drop your /dist directory onto Netlify Drop (invalid link)

Cloudflare Pages

Same as #1, but also supports directories (You know what to do, upload your /dist directory)

nginx

Edit your nginx.conf file to add the dist directory as the main directory, and index.html as your main page. Then run your nginx server, and voila! Your site is live on the configured domain!

Other hosting providers (mostly)

Add your /dist directory [Or the contents] to the site.

Other Webservers (mostly)

Configure the webserver to point to your dist directory as the root directory, and start the webserver.
